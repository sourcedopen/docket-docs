# Timeline

The **Timeline** tab on a ticket's detail page shows a unified, reverse-chronological view of all activity on that ticket — both manual comments and automatic system events.

## Comment Entries

Each comment entry shows:

- **Author name**
- **Type badge** (Update, Note, Response Received, Escalation, or Resolution)
- **Time ago** timestamp (e.g. "2 hours ago")
- **Body** text
- **Attachment links** (if files were attached)
- **Delete button** (visible only to the comment's author)

## Activity Log Entries

Automatic entries are created whenever a ticket is created, updated, or related records change. Each activity entry shows:

- **Event badge**:
  - Green — created
  - Blue — updated
  - Red — deleted
- **User** (or "System" for automated actions)
- **Up to 3 changed fields** with old → new values (e.g. `status: Draft → Submitted`)
- **Time ago** timestamp

## Order

All entries (comments and activity) are displayed **newest first** so the most recent events are always at the top of the tab.
